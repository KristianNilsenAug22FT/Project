Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

There are a number of practices in front-end development that are widely accepted. The practices are proven to be efficient. They include:

Sanitization of data
Make sure that the data or input your site accepts is in the right format. You can do this manually or use third-party solutions.

Secure authorization
Ensure that authorization is secure. Secure authentication of the user is important to make sure no one can hack into your site. You can also do this manually, but it is best that you use well known authorization solutions, like the industry-standard OAuth.

Use well know third-party apps
If you use third-party solutions for your site, make sure they are well-known and do research to make sure it is safe to use.

CI/CD
Protect changes on your site by using Continuous Deployment/Continuous Integration. With CI/CD every change you make to the site will have to go through security checks before delivering the changes to production.

Network security
Be wary of where you host the site. Do research into the hosting service before choosing. If the hosting service has bad network security your data could easily be compromised on the hosting service.

Timeout
Use automatic timeout for users. If the user has put in credentials or other info on the site, make sure the site times out the session after a given amount of time. This protects the user.
